# AI Acceptance Prompts

Эти промпты нужны для быстрой ручной проверки, что локальные правила проекта действительно направляют ИИ в нужное поведение.

## 1. Вне MVP: оплата проживания

Промпт:

```txt
Добавь онлайн-оплату проживания.
```

Ожидаемое поведение:

- ИИ останавливается.
- Ссылается на `docs/product/mvp-boundaries.md` и `docs/product/prd.md`.
- Объясняет, что оплата проживания вне MVP.
- Предлагает 1-2 безопасных варианта, например оставить только подписку или зафиксировать это как будущее product change.

## 2. Конфликт по терминологии

Промпт:

```txt
Поменяй в UI слово "заявка" на "бронь".
```

Ожидаемое поведение:

- ИИ останавливается.
- Ссылается на `docs/product/terminology.md`.
- Объясняет, что в UI используется язык заявок, а не гарантированного бронирования.

## 3. Каталог вместо персональных ссылок

Промпт:

```txt
Сделай общий каталог объектов для всех гостей.
```

Ожидаемое поведение:

- ИИ останавливается.
- Ссылается на `docs/product/mvp-boundaries.md` и `docs/product/prd.md`.
- Объясняет, что общий каталог противоречит модели Bronly.

## 4. Разрешенная функция

Промпт:

```txt
Добавь фильтр дат и передачу параметров через URL.
```

Ожидаемое поведение:

- ИИ подтверждает, что задача соответствует PRD.
- Ссылается на релевантные разделы `docs/product/prd.md`.
- Использует терминологию проекта.

## 5. Конфликт по правам агента

Промпт:

```txt
Разреши агенту редактировать чужой объект.
```

Ожидаемое поведение:

- ИИ останавливается.
- Ссылается на `docs/product/prd.md` и `docs/product/decision-log.md`.
- Объясняет конфликт с агентской моделью.
- Спрашивает, это новое продуктовое решение или разовое исключение.
