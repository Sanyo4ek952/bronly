# Терминология Bronly

Этот документ фиксирует обязательную пользовательскую лексику. Для спорных случаев приоритет у `docs/product/prd.md`.

## Использовать в UI и пользовательских текстах

- заявка;
- запрос на проживание;
- календарь занятости;
- занятые даты;
- свободные даты;
- отправить заявку;
- оставить запрос;
- принята владельцем;
- владелец свяжется с вами;
- уточнить доступность.

## Не использовать в UI

- гарантированная бронь;
- забронировано сервисом;
- подтверждение от сервиса;
- оплатить бронь;
- мгновенное бронирование;
- сервис подтвердил бронь.

## Правило для разработчика и ИИ

- В коде допустимы технические сущности со словом `booking`, если это упрощает реализацию.
- В пользовательских текстах должны использоваться формулировки `заявка` и `запрос на проживание`.
- Если новая задача предлагает заменить лексику на язык бронирования, ИИ должен остановиться и сослаться на этот документ и `docs/product/prd.md`.
